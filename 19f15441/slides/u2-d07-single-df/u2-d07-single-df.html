Select helpers

  • starts_with(): Starts with a prefix
  • ends_with(): Ends with a suffix
  • contains(): Contains a literal string
  • num_range(): Matches a numerical range like x01, x02, x03
  • one_of(): Matches variable names in a character vector
  • everything(): Matches all variables
  • last_col(): Select last variable, possibly with an offset
  • matches(): Matches a regular expression (a sequence of symbols/characters expressing a string/pattern to be searched for within text)

.footnote[ See help for any of these functions for more info, e.g. ?everything.]

summarise and group_by

summarise for summary stats

# mean average daily rate for all bookings
hotels %>%
  summarise(mean_adr = mean(adr)) #<<
# A tibble: 1 x 1
  mean_adr
     <dbl>
1     102.

–

.pull-left-wide[ .tip[ summarise() changes the data frame entirely, it collapses rows down to a single summary statistic, and removes all columns that are irrelevant to the calculation.]]

.tip[ summarise() also lets you get away with being sloppy and not naming your new column, but that’s not recommended!]

.pull-left[ ❌] .pull-right[ ✅]

group_by for grouped operations

# mean average daily rate for all booking at city and resort hotels
hotels %>%
  group_by(hotel) %>% #<<
  summarise(mean_adr = mean(adr))
# A tibble: 2 x 2
  hotel        mean_adr
  <chr>           <dbl>
1 City Hotel      105. 
2 Resort Hotel     95.0

Multiple summary statistics

summarise can be used for multiple summary statistics as well

hotels %>%
  summarise(
    min_adr = min(adr),
    mean_adr = mean(adr),
    median_adr = median(adr),
    max_adr = max(adr)
    )
# A tibble: 1 x 4
  min_adr mean_adr median_adr max_adr
    <dbl>    <dbl>      <dbl>   <dbl>
1   -6.38     102.       94.6    5400
